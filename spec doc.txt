Global engineering standards (put this at the top of every agent session)
Socket namespace/rooms: one Socket.IO namespace /classroom, clients join room session:<roomId> on connect. Never broadcast globally — always io.to(session:${roomId}).
Auth on socket: pass the existing JWT as a handshake auth token (socket.handshake.auth.token), verify with the same logic as verifyToken in server.cjs, attach { userId, name, role } to socket.data. Reject the connection if invalid.
Server-side source of truth: participant list, hand-raise state, mic/cam state, permissions, waiting room, and lock state live in an in-memory room registry (Map<roomId, RoomState>) on the server — not just on clients. Broadcast full or diffed state on every change so late joiners get correct state.
Event naming convention: noun:verb, present tense for client→server requests, past tense for server→client broadcasts. e.g. client emits hand:raise, server broadcasts hand:raised.
Every socket handler must: (1) verify the room exists, (2) verify the sender's role/permission for trainer-only actions, (3) update room state, (4) broadcast, (5) fire-and-forget an activity-log write (Module 9) via an internal event emitter — don't block the broadcast on the DB write.
REST vs Socket: session create/list/delete stays REST (already exists). Anything that needs to be instant and multi-user (hand raise, mute, permissions, participant removal, waiting room) is Socket-only. Activity logs are written via socket-triggered server-side calls, not client REST calls.
Reconnection: on client reconnect, re-join the room and request a full state sync (room:sync request → room:state response) rather than relying on missed broadcasts.
No new video/SFU infra assumption: use plain WebRTC mesh (RTCPeerConnection per participant pair) with Socket.IO purely as the signaling channel (webrtc:offer, webrtc:answer, webrtc:ice-candidate). Note in your PR if participant count in testing exceeds ~6, since mesh degrades beyond that — flag it, don't silently build an SFU (that's a separate infra decision, not part of this scope).
Code style: match existing .cjs CommonJS style in server/, existing functional-component + hooks style in src/. Keep new components under src/live-classroom/ (create the folder) instead of dropping everything in src/ root.
